角色设定

你是一个专业的前端工程师和儿童教育产品设计师。请帮我写一个单文件 HTML 交互式 Web 应用（包含所有 CSS 和 JS），用于小学一年级下册语文的“一类字”识字学习。

核心目标与受众

受众：小学一年级儿童（6-7岁），以形象记忆为主，注意力集中时间有限。

目标：建立“字形-字音-字义（图像）”的强关联，通过游戏化测验巩固记忆。

整体风格：可爱、欢快、色彩柔和，排版类似语文课本，按钮要大，适合触摸屏点击。使用 Tailwind CSS 进行样式设计。

数据与进度管理（防疲劳机制）

内置词库：将一年级下册语文课本的所有生字（约300字）作为默认字库，并自动去重。

精准图像字典：在 JS 中内置一个庞大的 Emoji 字典（如 '水': ['💧', '🌊', '💦']）。必须保证映射绝对准确，绝不能使用随机图标兜底（遇到没有图像的抽象字直接显示文字图标即可）。

分阶段学习：为了防止孩子疲劳，将生字按每组 15 个字切分。首页需要有一个“选择学习进度”的下拉菜单（例如：第1关 1~15字，第2关 16~30字）。选择关卡后，文本框自动填入这15个字，且允许家长手动增删修改。

核心功能模块

1. 沉浸式学习模式

界面布局：屏幕中央是一张大卡片，上面显示【汉字】（特大字号），汉字正上方标注【拼音】（带声调，使用 ruby 标签）。汉字下方显示代表该字含义的【图片/Emoji】（极大的尺寸，极具视觉冲击力）。

极简交互（直觉操作）：

点击汉字区域：播放该汉字的读音，并有轻微的缩放点击反馈。

点击图片区域：播放该汉字的读音，并且如果该字有多个不同的图片（如'水'有水滴和海浪），每次点击自动切换下一张图片。

去除多余的“发音”或“换图”按钮，只保留“上一个”、“下一个”按钮。

添加微小的提示文字引导（如“👆 点击汉字听发音”）。

2. 游戏化测验模式

包含两种测验类型：

A. 看字找图：给出一个汉字，从 4 个 Emoji 图片中选出对应的意思。注意：出题时必须严格过滤，只用“字典里有明确实体图片”的汉字出题，跳过抽象字。

B. 看字找音：给出一个汉字，从 4 个拼音中选出正确的读音。

测验交互：

选对时：发出正确提示音，按钮变绿，全屏撒花（使用 canvas-confetti），延迟1秒自动进入下一题。禁用其他按钮防止连点。

选错时：按钮左右抖动（Shake 动画），变红，语音提示“再想想”。

样式细节：严格禁用选项按钮的 hover 状态边框残留，必须加 focus:outline-none，防止在平板/手机触摸屏上点击后留下干扰颜色的边框。

结算：结束后准确显示“共完成了 X 道题”和“总得分”。

3. 音频与特效要求（极其重要）

语音朗读（TTS）：使用浏览器的 SpeechSynthesisUtterance。必须加入“打断重读”机制（window.speechSynthesis.cancel()），防止孩子手速过快连续点击导致声音排队堆叠。

全局背景音乐（BGM）：

必须有欢快的轻音乐作为背景音（音量调小至15%，不能盖过朗读声）。

考虑到浏览器的 autoplay 限制，请使用 Web Audio API（自动弹奏合成的轻音乐旋律），彻底放弃引入外部 mp3 链接，杜绝跨域和加载失败问题。

在用户第一次点击屏幕任何地方时，顺畅启动背景音乐。

顶部提供一个全局的 BGM 开关按钮（🎵/🔇）。

技术要求

必须整合为一个单文件 HTML。

响应式设计：完美适配手机、平板和 PC。

隐藏浏览器的默认滚动条，但保留内容的滚动功能。
